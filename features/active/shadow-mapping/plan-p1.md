# Phase Plan: Shadow Mapping — Phase 1: Hard Shadows

**Gate**: Feature Brief approved (Phase 1 gate)
**Roadmap alignment**: Phase 1 — Hard Shadows (see brief.md)
**Build target**: `cmake --build build --target renderer_app`
**Groomer Verdict**: PASS
**Revision**: r2 — addressed groomer defects 1 (T-2 vertex stride), 2 (T-6 acceptance), 3 (T-6 ShadowFrameParams per-draw application)

## Expected Outcome

Running `renderer_app` shows a shadow demo scene (selectable via ImGui) where a directional light from above-slightly-tilted casts hard shadows from spheres and cubes onto a flat plane and onto each other. The orbiting camera reveals shadows from multiple angles without acne or flickering. Standard `BlinnPhong` objects in the same scene are unaffected. The existing renderer demo remains intact.

## Fallbacks

- **T-3 (sg_make_view API mismatch)**: If `sg_view_desc.depth_stencil_attachment` does not exist in the pinned sokol version, check for `sg_attachments` / `sg_make_attachments` API (older sokol pattern) and adapt accordingly. Use `sg_begin_pass` with the attachment-based variant. Escalate to human if neither API is available.
- **T-4 (sampler2DShadow binding)**: If sokol-shdc rejects combined `texture2D + sampler` with `compare = SG_COMPAREFUNC_LESS` on the GL backend, fall back to manual PCF (4-sample manual depth comparison in FS) using a regular `sampler2D`. This is a softer shadow edge but functionally correct for Phase 1.
- **T-6 (ShadowFrameParams binding index)**: If the generated uniform binding index for `ShadowFrameParams` does not match `UB_shadow_depth_vs_params` (naming collision with shadow VS uniforms), use the sokol-shdc generated constants (`UB_blinnphong_shadowed_shadow_frame_params` or equivalent) rather than a hardcoded `2`.

## Human Checkpoint

**Run**: `cmake --build build --target renderer_app && ./build/renderer_app`

**Look for**:
1. ImGui tab bar shows "Shadow Demo" tab; clicking it switches to the shadow scene.
2. Hard shadow visible on the plane from at least the largest sphere (sphere A directly above).
3. Cube casts a recognizable hard shadow on the plane.
4. The magenta-tinted comparison sphere (BlinnPhong, no shadow receive) renders fully lit — no shadow darkening on it.
5. The transparent blue sphere renders with no shadow involvement (fully lit through its body).
6. Camera orbits: shadows move realistically with the static directional light; no per-frame shadow crawl or severe acne on lit surfaces.
7. "Show shadow map" ImGui checkbox (in Shadow Demo tab) reveals a greyscale depth texture in the top-right corner showing scene depth from light perspective.
8. Switching back to "Existing Demo" tab: the original demo renders identically to before this feature — no regression.
9. Console shows no sokol validation errors or GL errors.

**Pass**: All 9 criteria met. Shadows are hard-edged (pixelated edges are expected — PCF is Phase 3). Some minor acne on grazing surfaces is acceptable if confined to steep angles; zero acne on mostly-flat surfaces is required.

**Stop**: Any of the following — shadows absent entirely from the plane, severe acne covering broad illuminated surfaces, transparent/BlinnPhong objects incorrectly shadowed, existing demo visually broken, build exits non-zero, GL errors reported.
